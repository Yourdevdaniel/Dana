# User Flow

## Entrada

1. usuario abre a aplicacao
2. se nao ha sessao, entra na tela de login/cadastro
3. se a conta nao foi verificada, vai para a tela de verificacao
4. se a sessao e valida, carrega o dashboard

## Cadastro

1. preencher nome, email e senha
2. backend valida email unico e senha forte
3. backend envia email de verificacao
4. usuario confirma pelo link

## Login

1. email e senha
2. opcionalmente Google
3. caso a sessao expire, retornar para login

## Navegacao interna

- dashboard
- carteira
- transacoes
- categorias
- metas
- dividas
- contas fixas
- casal
- conquistas
- perfil
- rede
